Field Expert: directly create a field for a document. Most users should use one of the sugar subclasses above.

(*1) If you also need to store the value, you should add a separate StoredField instance.

(*2) Finding all documents within an N-dimensional shape or range at search time is efficient. Multiple values for the same field in one document is allowed.
